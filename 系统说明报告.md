# 筑纹杯 CupForge — 系统说明报告

**版本**：v2.0  
**系统类型**：Web 应用（浏览器访问）  
**适用对象**：使用者、课程答辩、项目交付说明  

---

## 1. 系统概述

**筑纹杯 CupForge** 是一套面向 **工业建造文化** 的纹样设计与 3D 杯具定制系统。用户上传工业建筑类照片（如厂房、大桥、塔吊等），系统通过计算机视觉自动提取线稿、色板与主题标签，再在前端完成纹样阵列编辑与 3D 杯型预览，最终可导出 **STL** 文件用于 3D 打印或后续建模。

系统采用 **前后端分离** 架构：后端负责图像处理与部分 3D 导出，前端负责交互编辑、实时预览与浮雕 STL 生成。部署到 Render 等云平台后，可通过公网链接直接使用，无需安装客户端。

## 2. 系统功能

### 2.1 图像上传与智能分析

| 功能 | 说明 |
|------|------|
| 图片上传 | 支持常见图像格式（jpg、png 等），单次上传一张 |
| 工业主题识别 | 自动识别 10 类工业主体：厂房、大桥、高楼、起重机、吊车、烟囱、管道、钢架、塔吊、脚手架；低置信度时显示「工业建筑」 |
| 主色板提取 | KMeans 聚类提取 5 种主色及占比，供后续配色使用 |
| 视觉形态生成 | 一键生成 5 种形态：原图、线稿、像素、半调、剪影 |
| 参数实时重算 | 线稿细节、像素粒度可调，后端按会话重新计算形态 |

### 2.2 纹样编辑

| 功能 | 说明 |
|------|------|
| 随机方案 | 贴图流程提供 6 套随机纹样方案，可一键应用 |
| 取样框编辑 | 可拖动、缩放、旋转取样区域 |
| 取样形状 | 矩形、圆形、三角形、菱形、梯形 |
| 阵列模式 | 8 种：普通平铺、水平/垂直镜像、旋转万花筒、错位砖墙、移步互换、四轴中心对称、比例渐变 |
| 配色模式 | 自动色板、单色调、自由选色 |
| 实时预览 | 左侧取样区 + 右侧阵列结果同步更新 |

### 2.3 3D 杯具定制

| 功能 | 说明 |
|------|------|
| 杯型预设 | 5 种：直杯、茶杯、酒杯、笔筒、花瓶 |
| 轮廓编辑 | 贝塞尔曲线控制点捏制杯壁外形 |
| 尺寸参数 | 杯壁厚度、杯高可调 |
| 材质与颜色 | 5 种材质预设；杯身支持纯色 |
| 贴图区域 | 可在杯身展开图上划定多块贴图区域；**允许不贴任何纹样**（整杯为杯体色） |
| 3D 预览 | Three.js 轨道旋转、重置视角、全屏 |

### 2.4 浮雕与导出

| 功能 | 说明 |
|------|------|
| 浮雕预览 | 线稿纹样转为高度图，沿杯壁法线几何位移 |
| 浮雕参数 | 凸起/凹陷方向、浮雕深度可调 |
| 浮雕区域 | 可划定多块浮雕生效区域 |
| STL 导出 | 见下表 |

| 使用场景 | 导出内容 | 生成方式 |
|----------|----------|----------|
| 贴图流程 · 3D 预览页 | 光滑杯壁 STL/OBJ | 后端 API（不含平面纹样几何） |
| 浮雕流程 · 浮雕工作室 | 带浮雕细节的 STL | 前端生成（**可 3D 打印纹样**） |

## 3. 使用流程

### 3.1 总体流程

```
打开系统 → 上传工业建筑照片 → 选择「贴图」或「浮雕」→ 编辑纹样与杯型 → 3D 预览 → 导出 STL
```

### 3.2 贴图流程（平面纹样贴附杯壁）

适合：希望杯身呈现完整阵列纹样、以视觉贴图效果为主的场景。

1. **首页** — 点击「开始设计」
2. **上传页** — 点击「+」上传照片，等待识别主题显示
3. **选择工艺** — 点选「贴图」，再点「开始生成」
4. **方案页** — 浏览 3 套随机方案，点击心仪方案进入编辑
5. **纹样编辑页**
   - 调整取样框位置、大小、旋转
   - 切换视觉形态（线稿/像素/半调等）
   - 选择阵列模式与行列数
   - 调整配色与线稿粗细
   - 满意后点击「进入 3D」
6. **3D 预览页（贴图）**
   - 选择杯型预设（如笔筒、茶杯）
   - 调节壁厚、高度，可编辑杯壁轮廓
   - 在「贴图区域」调整纹样覆盖范围（可删至 0 块，整杯不贴图）
   - 设置非贴图区域颜色
   - 旋转查看 3D 效果
   - 点击「导出」下载光滑杯型 STL

### 3.3 浮雕流程（可打印几何浮雕）

适合：需要将纹样转化为杯壁凹凸、用于 3D 打印的场景。

1. **上传页** — 上传照片后选择「浮雕」，点击「开始生成」
2. **浮雕工作室**（跳过方案页，直接进入）
   - 左侧：线稿纹样编辑（取样、阵列、线宽、细节等）
   - 中间：纹样阵列预览
   - 右侧：3D 浮雕预览
   - 调整杯型、浮雕深度与方向、浮雕区域
3. **导出** — 点击「导出浮雕 STL」，下载 `cupforge_cup_relief.stl`

### 3.4 使用建议

- 上传图片建议 **小于 2MB**，主体清晰、工业结构明显，识别效果更好
- 在线版（Render 免费实例）长时间未访问会休眠，首次打开需等待约 30～60 秒
- 服务重启后需 **重新上传** 图片（会话保存在服务器内存中）
- 3D 预览与复杂编辑建议使用 **电脑 + Chrome/Edge**

## 4. 核心技术

### 4.1 技术架构

```
┌─────────────────────────────────────────┐
│              浏览器（用户端）              │
│  React + TypeScript + Canvas + Three.js │
└─────────────────┬───────────────────────┘
                  │ HTTP /api/*
┌─────────────────▼───────────────────────┐
│           FastAPI 后端（服务端）          │
│  OpenCV · scikit-learn · trimesh        │
└─────────────────────────────────────────┘
```

生产环境（Render）：Docker 单容器同时托管前端静态文件与后端 API，用户访问同一域名。

### 4.2 后端核心技术

| 模块 | 路径 | 技术要点 |
|------|------|----------|
| 图像预处理 | `backend/pipeline/preprocess.py` | 缩放、双边滤波、CLAHE 增强、Canny 边缘、轮廓过滤 |
| 色板提取 | 同上 | KMeans 聚类，过滤高亮背景像素 |
| 视觉形态 | `backend/pipeline/expressions.py` | 线稿、像素量化、半调网屏、剪影形态学 |
| 主题识别 | `backend/pipeline/industrial_recognize.py` | 边缘密度、Hough 线方向、空间分布启发式分类 |
| 杯型网格 | `backend/pipeline/cup_mesh.py` | B 样条轮廓 + 绕 Y 轴旋转生成双层壁厚网格 |
| API 服务 | `backend/main.py` | REST 接口、会话管理、静态资源托管 |

**主要依赖**：OpenCV、scikit-learn、NumPy、SciPy、FastAPI、trimesh、uvicorn  

**参数配置**：算法阈值、杯型预设等集中在 `backend/config.py`。

### 4.3 前端核心技术

| 模块 | 路径 | 技术要点 |
|------|------|----------|
| 纹样引擎 | `frontend/src/engine/patternEngine.ts` | 取样裁剪、8 种阵列平铺、配色映射 |
| 取样交互 | `frontend/src/engine/sampleRegion.ts` | 取样框拖动/缩放/旋转 |
| 杯型网格 | `frontend/src/engine/cupMesh.ts` | 与后端一致的回转体网格生成 |
| 浮雕位移 | `frontend/src/engine/cupRelief.ts` | 高度图双线性采样 + 法线方向顶点位移 |
| STL 导出 | `frontend/src/engine/cupStlExport.ts` | 前端二进制 STL 写入 |
| 3D 渲染 | `frontend/src/components/Cup3DPreview.tsx` | Three.js 场景、贴图、PBR 材质、轨道控制 |
| 页面路由 | `frontend/src/App.tsx` | 单页状态机，管理全局会话与编辑状态 |

**主要依赖**：React 18、TypeScript、Vite、Three.js  

### 4.4 接口一览

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/health` | 服务健康检查 |
| GET | `/api/config` | 获取前端配置与枚举 |
| POST | `/api/upload` | 上传图片并处理 |
| POST | `/api/reprocess` | 按会话重算视觉形态 |
| POST | `/api/cup/export/stl` | 导出光滑杯型 STL |
| POST | `/api/cup/export/obj` | 导出光滑杯型 OBJ |

## 5. 运行方法

### 5.1 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows / macOS / Linux |
| Python | 3.10+（推荐 3.11） |
| Node.js | 18+（推荐 20） |
| 浏览器 | Chrome、Edge 等现代浏览器 |

### 5.2 本地开发运行

**第一次使用需安装依赖：**

```cmd
# 后端
cd backend
python -m venv venv
venv\Scripts\activate.bat
pip install -r requirements.txt

# 前端
cd frontend
npm install
```

**日常启动（需两个终端）：**

```cmd
# 终端 1 — 后端（端口 8000）
cd backend
venv\Scripts\activate.bat
python main.py

# 终端 2 — 前端（端口 5173）
cd frontend
npm run dev
```

浏览器访问：**http://localhost:5173**

> 开发模式下，前端通过 Vite 代理将 `/api` 请求转发到 `localhost:8000`。

### 5.3 本地生产模式验证（可选）

验证「单端口同时提供网页与 API」是否正常：

```cmd
cd frontend
npm run build

cd ..\backend
venv\Scripts\activate.bat
set PORT=8000
python main.py
```

浏览器访问：**http://localhost:8000**

### 5.4 在线部署运行

系统已支持 Docker + Render 部署，详细步骤见 **[DEPLOY.md](./DEPLOY.md)**。

**简要流程：**

1. 代码推送到 GitHub 仓库  
2. 在 Render 创建 Web Service，Runtime 选 **Docker**，Instance 选 **Free**  
3. 部署完成后获得公网地址，例如 `https://cupforge.onrender.com`  
4. 将链接分享给他人即可使用  

**更新线上版本：**

```cmd
git add .
git commit -m "更新说明"
git push
```

Render 检测到 push 后自动重新构建部署（约 5～15 分钟）。

## 6. 系统限制与说明

| 项目 | 说明 |
|------|------|
| 会话存储 | 上传会话保存在服务器内存，重启或冷启动后需重新上传 |
| 无用户系统 | 不支持登录、历史作品保存 |
| 贴图 STL | 贴图流程导出的 STL 为光滑杯壳，平面纹样仅用于 3D 渲染贴图 |
| 浮雕 STL | 浮雕流程导出的 STL 包含纹样凹凸几何，适合 3D 打印 |
| 页面状态 | 刷新浏览器会回到首页，未保存的编辑状态会丢失 |
| 在线性能 | 免费云实例内存有限，大图处理可能较慢 |
