## 2. 核心用户体验链路 (User Flow)

### 阶段 0：进入系统

**用户目标**：确认工具可用，准备输入素材  

**系统行为**：

- 加载后端配置（Canny 默认值、形态列表、阵列选项等）
- 状态栏显示：「系统就绪。请上传图像或选取测试样本。」
- 若后端未连接：「后端服务未连接。请先启动 Python API 服务。」
- 右侧三处画布均为空状态占位文案；多数控件处于禁用或无效状态

---

### 阶段 1：输入图像

**用户目标**：提供一张工业建造类照片作为纹样来源  

**路径 A — 本地上传**：左侧「图像源」选择文件  
**路径 B — 测试图库**：点击某类别缩略图，等效于上传  

**系统行为**：

- 状态栏：「正在执行图像预处理与特征提取…」
- 后端完成：边缘检测、5 种视觉形态生成、K-means 提取 5 主色
- 完成后：
  - 左上显示原图 + 居中取样框（默认占图宽 25%）
  - 右上显示当前选中形态的预览
  - 下方显示平铺纹样预览
  - 左侧色彩映射区块出现
  - 状态栏报告形态数量与主色数量

---

### 阶段 2：调整预处理与视觉形态

**用户目标**：找到最满意的线条风格与视觉表达方式  

**可执行操作**：

1. 拖动「线条细节程度 / 过滤强度」→ 300ms 防抖后请求后端重算全部 5 种形态
2. 切换「视觉形态」下拉 → 即时切换预览源图（无需等待后端）
3. 按形态显示专属滑块：
   - 线稿：线条粗细（**前端即时**，无网络请求）
   - 像素化 / 光晕：程度滑块（**300ms 防抖 + 后端重算**）

**用户感知**：形态切换快；线稿加粗即时；Canny/像素/光晕调整有短暂计算延迟

---

### 阶段 3：定义取样区域

**用户目标**：框选纹样单元，决定平铺的基本「砖块」  

**可执行操作**（仅在已上传后，两处画布同步）：

- **移动**：拖拽取样框内部
- **缩放**：拖拽 8 个黄色角/边控制点（最小 20×20 px）
- **旋转**：拖拽框上方蓝色圆点（带连接线指示）
- **改形状**：取样形状下拉（矩形/圆/三角/菱形/梯形）→ 裁剪路径即时变化
- **改比例**：取样框比例滑块 → 以图像中心重置框大小，保留当前旋转角

**系统反馈**：任意取样框变更 → 下方平铺预览**即时重绘**（纯前端）

---

### 阶段 4：配置阵列与色彩

**用户目标**：构建无缝纹样效果，并按品牌/套印需求着色  

**阵列操作**：

- 选择阵列模式、列数、行数 → 平铺预览即时更新
- 万花筒模式额外显示「分段数」滑块
- 比例渐变模式额外显示「渐变方向」下拉

**色彩操作**：

- 勾选「启用色彩映射」→ 平铺预览按所选模式重新着色
- 三种映射模式切换 → 控件区动态展开/收起
- 所有色彩参数变更 → 平铺预览即时更新

---

### 阶段 5：导出成果

**用户目标**：下载可用于生产或 3D 制作的文件  

**操作**：

1. 选择「贴图 PNG」或「浮雕 Heightmap」
2. 点击「下载导出文件」

**系统行为**：

- 贴图：直接下载当前平铺预览的高分辨率 Canvas
- 浮雕：前端转灰度 → 尝试后端增强 → 下载 PNG Heightmap（白=凸起，黑=凹陷）
- 导出中按钮文案变为「导出中…」，完成后恢复

### 3.2 左侧控制栏

#### 区块 A：输入与预处理

| 控件 | 类型 | 枚举/范围 | 联动规则 |
|------|------|-----------|----------|
| 图像源 | File Upload | 所有 `image/*` | 选中即触发上传流程；重复选同文件需重新选择 |
| 线条细节程度 | Slider | 10–200，默认 50 | 与过滤强度成对触发后端重算；300ms 防抖 |
| 线条过滤强度 | Slider | 50–400，默认 150 | 同上 |
| 重算提示 | 内联文案 | — | 上传中或重算中显示「正在重算视觉形态…」 |

**用户友好标签映射**（Canny 算法对用户隐藏）：

- 细节程度 ↓ = 保留更多细线
- 过滤强度 ↑ = 剔除更多噪点

---

#### 区块 B：视觉表现形态

| 控件 | 类型 | 完整枚举 |
|------|------|----------|
| 视觉形态 | Dropdown | ① 纯粹线稿 ② 复古像素化 ③ 报纸半调 ④ 高反差剪影 ⑤ 光晕弥散 |

| 条件形态 | 子控件 | 类型 | 范围 | 响应方式 |
|----------|--------|------|------|----------|
| 纯粹线稿 | 线条粗细 | Slider | 1–8 | **前端即时**加粗线稿 |
| 复古像素化 | 像素化程度 | Slider | 2–32 | 300ms 防抖 → 后端重算 |
| 光晕弥散 | 光晕弥散程度 | Slider | 1–25（步进 0.5） | 300ms 防抖 → 后端重算 |
| 报纸半调 / 高反差剪影 | — | 无专属控件 | — | — |

**注意**：下拉菜单标签当前为英文 `Expression Form`，设计时可改为中文「视觉形态」。

**禁用规则**：未上传图像前，视觉形态下拉为 Disabled。

---

#### 区块 C：取样与阵列

| 控件 | 类型 | 完整枚举 / 范围 |
|------|------|-----------------|
| 取样形状 | Dropdown | 矩形 · 圆形 · 三角形 · 菱形 · 梯形 |
| 阵列模式 | Dropdown | ① 普通平铺 ② 水平镜像 ③ 垂直镜像 ④ 旋转万花筒 ⑤ 错位砖墙 ⑥ 移步互换 ⑦ 四轴中心对称 ⑧ 比例渐变 |
| 取样框比例 | Slider | 0.10–0.60 |
| 列数 | Slider | 2–8 |
| 行数 | Slider | 2–8 |
| 万花筒分段数 | Slider（条件） | 3–12，仅「旋转万花筒」时显示 |
| 渐变方向 | Dropdown（条件） | 垂直方向（上→下）· 水平方向（左→右），仅「比例渐变」时显示 |

**禁用规则**：取样形状、阵列模式、渐变方向 — 未上传前 Disabled。  
**取样框比例、列数、行数、万花筒分段数** — 上传前可操作但无可见效果。

**实时性**：列/行/阵列/形状/比例变更 → **平铺预览即时更新**（纯前端，无防抖）。

---

#### 区块 D：色彩映射

**出现条件**：上传完成且已提取色卡（5 色）后，整个区块才渲染。

| 控件 | 类型 | 说明 |
|------|------|------|
| 启用色彩映射 | Checkbox | 默认关闭；关闭时平铺预览显示形态原色 |
| 映射模式 | Dropdown | 自动吸附 · 基于原图色卡调色 · 自由选择颜色 |

**模式 A — 自动吸附**

- 无额外控件
- 逻辑：每个像素替换为 K-means 色卡中**欧氏距离最近**的颜色
- 效果：有限色数套印感，保留原图色相分布

**模式 B — 基于原图色卡调色**

- 色卡多选按钮 × 5（Toggle，至少保留 1 色）
- 饱和度 0–200%
- 对比度 50–200%
- 亮度偏移 -50–50
- 逻辑：仅在**已选色卡子集**中找最近色 → 再应用饱和/对比/亮度调整
- 效果：可控的多色套印，可偏冷/偏暖/高对比

**模式 C — 自由选择颜色**

- 5 个 Color Picker（色阶 1–5），各显示 Hex 值
- 逻辑：按像素**亮度**分 5 档，映射到对应自定义色
  - 最暗 → 色阶 1，最亮 → 色阶 5
- 平铺背景色：取色阶 1 作为阵列底色
- 效果：完全脱离原图色相，适合品牌色定制

**提取色卡（只读）**

- 5 个色块，hover 显示 Hex，块内显示占比（如 `32%`）
- 不可点击编辑，仅供参考

**实时性**：所有色彩映射变更 → **平铺预览即时更新**

---

## 4. 界面状态定义 (State Management)

### 4.1 空状态 (Empty State)

```
┌─────────────────────────────────────────────────────────┐
│  顶栏：CupForge                                    │
├──────────────┬──────────────────────────────────────────┤
│ 左栏         │  上左：空状态占位                         │
│ · 可上传     │  上右：空状态占位                         │
│ · Canny 可用 │  ─────────────────────────────────────── │
│ · 形态禁用   │  平铺预览：大空状态占位                    │
│ · 无色彩区块 │                                          │
│ · 导出禁用   │  状态栏：系统就绪…                        │
│ · 图库可见   │                                          │
└──────────────┴──────────────────────────────────────────┘
```

**设计要点**：

- 空画布需有明确引导（上传 or 选测试图）
- 禁用控件应视觉区分（当前仅 opacity + cursor）
- 色彩映射区块完全隐藏，避免用户困惑

---

### 4.2 加载状态 (Loading)

| 场景 | 现有表现 | 设计建议 |
|------|----------|----------|
| 首次上传 | 状态栏文字 + 左栏重算提示 | 可加全局轻量 Loading 遮罩或进度条 |
| Canny/像素/光晕重算 | 状态栏「正在重算…」+ 左栏提示 | 平铺预览可保留上一帧 + 右上角 Spinner |
| 测试图库加载 | 图库底部小字「加载中…」 | 缩略图区 Skeleton 占位 |
| 导出 Heightmap | 按钮文案「导出中…」+ Disabled | 按钮内 Spinner |

**重要**：加载期间**不会**阻止用户继续拖动取样框或切换阵列（除导出按钮外）。

---

### 4.3 报错状态 (Error State)

| 错误类型 | 触发 | 现有反馈 |
|----------|------|----------|
| 后端未连接 | 启动时配置请求失败 | 状态栏红色系文案提示 |
| 非图片文件 | 后端拒绝 | 状态栏「错误: …」 |
| 上传/重算失败 | 网络或处理异常 | 状态栏「错误: …」 |
| 测试图不存在 | 图库目录为空 | 类别内灰色辅助文案（非弹窗） |
| Heightmap 后端失败 | 静默降级 | 使用前端灰度版本，用户无感知 |

**设计建议**：

- 错误应脱离状态栏，使用 Toast / Inline Alert 组件
- 区分「可恢复错误」与「需重新上传错误」
- 后端断开时，左栏顶部 persistent banner

---

### 4.4 完整态 (Active Session)

- 三画布均有内容
- 左栏全部控件可用（除仍 Disabled 的逻辑控件）
- 色彩映射 + 提取色卡可见
- 导出按钮可用（平铺预览渲染完成后）

## 5. 现存痛点与 UI 优化建议 (Design Tips)

### Tip 1：左栏信息过载 — 建议「分阶段折叠面板」

当前 6 个区块线性堆叠，纵向超过一屏，用户需频繁滚动才能从「上传」跳到「导出」。  

**建议**：

- 采用 Accordion / Step Tabs 结构：`① 输入` → `② 形态` → `③ 取样` → `④ 色彩` → `⑤ 导出`
- 上传完成后自动展开「形态」并收起「图库」
- 测试图库默认折叠，作为「高级 / 演示入口」

---

### Tip 2：工作区层级倒置 — 平铺预览应成为视觉主角

当前上段双画布仅占 ~280px，最大面积被深色空白包裹；用户核心关注点是**平铺纹样**，却挤在底部。  

**建议**：

- 改为「左栏控制 + 中央大平铺预览（70%+ 高度）」
- 原图 / 形态预览缩小为可切换的 Tab 或画中画（PiP），或合并为单一「取样编辑器」
- 平铺预览区增加棋盘格透明背景，便于判断无缝衔接

---

### Tip 3：条件控件缺乏上下文 — 形态专属参数应「贴靠形态选择器」

线条粗细 / 像素化 / 光晕三个滑块分散在形态下拉下方，用户切换形态时容易忽略新出现的控件。  

**建议**：

- 形态选择改为**卡片式 Radio Group**（5 张带缩略图示意）
- 选中卡片下方 inline 展开该形态专属参数
- 未选中形态参数完全隐藏，避免布局跳动（用固定 min-height 容器过渡）

---

### Tip 4：取样框交互缺乏引导 — 首次使用需 Onboarding

旋转蓝色圆点的说明只是一行小字，8 控制点 + 旋转手柄对非专业用户不直观。  

**建议**：

- 首次上传后显示 3 步 Coach Mark：移动 → 缩放 → 旋转
- 取样框 hover 时在框旁显示尺寸/角度 Tooltip
- 考虑增加「重置取样框」快捷按钮

---

### Tip 5：实时 vs 延迟反馈不统一 — 需视觉区分「即时」与「计算中」

线稿加粗、阵列切换 = 即时；Canny/像素/光晕 = 300ms 后后端计算。用户无法感知差异。  

**建议**：

- 即时参数：滑块旁标注「实时」标签（绿色）
- 后端参数：滑块旁标注「重算」标签，拖动时显示 debounce 进度条或 preview 半透明遮罩
- 重算完成后平铺预览做一次 subtle flash 或 check 动画，确认已更新
